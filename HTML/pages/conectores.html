Conectores (Inbound & Outbound)

Rutas Específicas para Tráfico Interno y Perimetral

Propósito Central

Los Conectores definen rutas específicas y reglas de confianza para que Exchange Online pueda comunicarse de forma controlada con servidores, dispositivos o sistemas de seguridad fuera del flujo de correo estándar.

1. Conectores de Recepción (Inbound)

Definen cómo Exchange Online debe aceptar el tráfico proveniente de sistemas internos, como servidores de aplicaciones o impresoras multifunción, para que no sean tratados como spam.

Configuración y Ventajas Clave

Confianza Basada en IP

Se configura especificando la dirección IP de origen de la red local. Esto crea una confianza implícita que no requiere autenticación (ideal para impresoras y dispositivos "cacharros").

Relay de Correo (Opción 3)

Permite que el sistema envíe correos aunque la dirección de origen no tenga un buzón asignado (notificaciones@dominio.com), siempre que el dominio esté aceptado. Esto es la mejor opción para aplicaciones internas.

Este conector utiliza el puerto 25 estándar y se salta las autenticaciones, gracias a la confianza en la IP de origen.

2. Conectores de Envío (Outbound)

Definen una ruta específica y no estándar para que el correo salga de Exchange Online hacia el exterior.

Uso Principal: Alteración del Flujo Saliente

  • Dispositivos de Seguridad Perimetral: Se utilizan para forzar que el correo saliente pase por un dispositivo de seguridad externo (ej. Barracuda) antes de ser entregado a Internet.

  • Rutas Específicas: Permiten dirigir el tráfico a una IP específica o a un servidor de correo particular de un socio, alterando la ruta pública normal.

3. Conector vs. Regla de Transporte

La Gran Diferencia: Apertura de Puertas

Una Regla de Transporte actúa sobre el contenido del correo ya en tránsito (añadir disclaimer, bloquear).

Un Conector de Recepción abre una segunda puerta que acepta conexiones desde una IP local confiable y está sujeta a menos restricciones Antispam. Solo el Conector permite hacer Relay (que un sistema sin buzón envíe correo a terceros).
