Gestión de Usuarios Activos (Azure AD)

Identidad, Licenciamiento y Seguridad en la Consola de Administración

1. El Eje Licencia-Funcionalidad

La lista de Usuarios Activos es la representación directa de los objetos de usuario en Azure AD y actúa como la lista maestra de identidades que pueden acceder a la plataforma.

Asignación de Licencias

Un usuario solo genera costo y activa funcionalidades (Exchange, Teams, SharePoint) cuando se le asigna una licencia.

Gestión de Sublicencias

Se pueden activar/desactivar servicios específicos dentro de un paquete de licencias para, por ejemplo, introducir herramientas progresivamente.

Configuración del UPN

Se modifica el Nombre Principal de Usuario (UPN), que es el nombre de inicio de sesión. Permite seleccionar el dominio correcto si la organización tiene varios.

Alias de Correo Electrónico (SMTP)

Se configuran alias de correo (direcciones SMTP). Tradicionalmente, solo se envía desde la dirección principal, pero se anticipa un cambio para enviar desde cualquiera de los alias.

⚠️ Nota: La interfaz web no permite obtener un listado completo de todos los alias fácilmente; para tareas masivas o de auditoría, se necesita PowerShell.

2. Permisos y Delegación de Buzones

Tríada de Permisos (Delegación)

Acceso Completo (Full Access)

Permite abrir el buzón, leer y administrar correos.

Enviar Como (Send As)

Envía correos con la identidad del buzón (útil para asistentes).

Enviar en Nombre de (Send on Behalf of)

Envía correos, pero muestra que fue enviado por un delegado.

Otras Configuraciones Clave

  • Reenvío de Correo: Configuración automática a otra dirección. Es considerada una acción peligrosa y común en ataques de compromiso, por lo que debe ser monitoreada.
  • Respuestas Automáticas: Configuración rápida de mensajes de "Fuera de la oficina" con distinción entre usuarios internos y externos.
  • Ocultar de la GAL: Permite ocultar al usuario de la Lista Global de Direcciones (GAL).

3. Seguridad y Gestión Masiva

Suspensión por Litigio (Litigation Hold)

Requiere licencia avanzada. Garantiza que todos los datos (incluyendo eliminaciones) queden retenidos legalmente y nunca se borren permanentemente.

Cerrar Sesiones Forzado

Medida de seguridad crítica en caso de brecha: fuerza el cierre de todas las sesiones abiertas por el usuario para bloquear el acceso inmediatamente.

Gestión Masiva con PowerShell

Para tareas masivas (creación, modificación, asignación de licencias), se utiliza PowerShell, importando datos desde archivos CSV para procesar a múltiples usuarios a la vez con bucles (for each).
