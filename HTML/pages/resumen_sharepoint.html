SharePoint en Microsoft 365: De Clásico a Moderno

El Repositorio Fundamental para Archivos y Colaboración

I. El Rol Central de SharePoint

SharePoint Online es una de las tres cargas de trabajo principales de Microsoft 365 (junto con Exchange Online y Teams). Su propósito es ser el repositorio troncal para la documentación y archivos de la organización, resolviendo las limitaciones de las carpetas compartidas tradicionales.

II. La Naturaleza Fundamental: Todo son Listas

SharePoint = Base de Datos SQL

El concepto fundamental es que SharePoint es un conjunto de tablas de una base de datos SQL. Incluso los archivos que usted sube son tratados como elementos dentro de una lista.

III. Evolución: Arquitectura Clásica vs. Moderna

El modelo moderno es el enfoque recomendado por Microsoft para evitar la rigidez y la complejidad de los permisos heredados del modelo clásico.

Modelo Clásico (Jerárquico)

  • Estructura: Jerarquía estricta de sitios y subsitios (similar a carpetas).
  • Permisos: Herencia compleja, muy detallados y granulares (editor, revisor).
  • Rigidez: Diseño y estructura difíciles de modificar una vez implementados.

Modelo Moderno (Plano y Flexible)

  • Estructura: Sitios independientes (islas) sin jerarquía estricta.
  • Navegación: La jerarquía visual se logra editando los menús (vínculos flexibles).
  • Permisos: Simplificados (dentro o fuera) para evitar errores.

Tipos de Sitio Modernos Recomendados

Sitio de Colaboración

Todos editan y modifican contenido. Se crea automáticamente al formar un Grupo de Office 365 o un Team.

Sitio de Comunicación

Orientado a la difusión (one-to-many). Un grupo pequeño edita, el resto visualiza (Ideal para Intranets).

IV. Doctrina Moderna y Administración

La Doctrina: Colaboración vía Teams

Filosofía de los Contenedores

Se debe trabajar con contenedores (Sitios o Teams) como la unidad de aislamiento de permisos. Si un grupo necesita acceso, se les da acceso completo al contenedor. Se evita la configuración detallada de permisos de carpetas (NTFS).

La forma más recomendada para que los usuarios interactúen con los archivos es a través de la interfaz sencilla y unificada de Teams.

Administración General (Panel de SharePoint)

  • Sitios Activos: Gestión de todos los sitios creados y monitorización del uso de almacenamiento (ej. límite por defecto: 1 TB + 1 GB por usuario).
  • Directivas (Policies): Establecimiento de reglas globales de compartición externa (limitar dominios, desactivar compartición anónima).
  • Personalización: Las propiedades administrativas (cuota, pertenencia a Hub Site) se manejan aquí; las ediciones de diseño se hacen en la propia interfaz del sitio.
