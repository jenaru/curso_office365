Roles de Administración y Granularidad de Permisos

Separación de Deberes y Control de Acceso Basado en Roles (RBAC)

1. El Concepto de Granularidad

Los roles de administración sirven para separar las tareas de administración. En lugar de que todos los administradores tengan el mismo acceso a todas las configuraciones (administradores globales), los roles permiten aplicar permisos de forma más granular para tareas específicas.

Tipos de Acceso

Administrador Global

Tienen todos los permisos sobre la plataforma. Aunque puedan no tener ciertos permisos de forma predeterminada (como abrir casos legales), tienen la capacidad de autootorgárselos.

Roles Especializados

Permiten asignar funcionalidades específicas (ej. Administrador de Facturación). Fomentan la seguridad por separación de funciones.

2. Roles en el Contexto del Correo Electrónico

El sistema de correo Exchange (la base de Exchange Online) fue el primer producto que utilizó un sistema de control de acceso muy granular llamado RPAC (Role-Based Access Control).

Granularidad Extrema (RPAC)

En Exchange, se puede definir un rol de administración diciéndole:

  1. Qué comandos de Power Shell van a poder utilizar.
  2. Con qué parámetros específicos.
  3. Para qué conjunto de usuarios (un subconjunto del directorio).

Ejemplo: Un rol podría limitar al administrador a cambiar solo direcciones de correo electrónico para los usuarios del departamento de marketing.

Roles Legales/Compliance: Existe una división crítica entre el rol de administración y el rol legal. Los profesionales legales necesitan acceso al centro de cumplimiento (Compliance) para tareas como realizar un descubrimiento legal (eDiscovery), sin tener acceso al panel de seguridad general.

3. Directivas de Paquetes de Roles (Teams)

En entornos complejos como Teams, asignar configuraciones uno por uno puede ser tedioso. Para solucionar esto, existen los:

Paquetes de Directivas (Policy Packs)

Un paquete de directiva es una configuración estandarizada que agrupa múltiples directivas para un perfil específico. Permite asignar todas las configuraciones necesarias de golpe.

Directiva de Chat Directiva de Llamadas Directiva de Reuniones Directiva de Aplicaciones ...y más

Ejemplo: Un paquete para el rol "Profesor" que activa todas las directivas de comunicación y desactiva ciertas aplicaciones externas.

En Resumen

El sistema de roles en Microsoft 365 está diseñado para ofrecer una separación de funciones (separar al administrador del abogado, por ejemplo) y una granularidad máxima, especialmente en Exchange y Teams, donde se pueden crear configuraciones muy específicas para subconjuntos de usuarios.
